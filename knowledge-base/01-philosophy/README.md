# Philosophy

The foundational documents about what TomorrowLabs believes about data, communities, and the work.

## Documents in this folder

| Document | What it is |
|----------|-----------|
| `data-philosophy-v1.md` | Markdown summary of the philosophy + link to the HTML artifact |
| `pressure-test.md` | Markdown summary of the adversarial critique + link to HTML |

## Why this folder is first

Everything else in TomorrowLabs's data work is justified by what's in this folder. The architecture exists to enforce the philosophy. The schemas encode the commitments. The decision protocols apply the values.

If you have to choose what to read first, read the philosophy.

## The most important commitments

From the philosophy (seven beliefs):

1. Languages exist in hierarchies, not lists
2. Confidence is a first-class data type
3. Every fact resolves to a source
4. Glottolog as tool, not truth (academic frameworks support community sovereignty, don't replace it)
5. Time-series tells the truth that snapshots cannot
6. The archive outlasts the organization
7. Community-contributed data belongs to the community

From the pressure-test (six critiques accepted as valid):

1. "Tool not truth" needs operational mechanisms, not just sentences
2. Benefit-sharing formulas must be specific and audited
3. TomorrowLabs is honestly a hybrid (commercial + research), not one or the other
4. Multi-generational commitments require irrevocable archive partnerships
5. Confidence labels don't fix biased data coverage
6. Community absence from the philosophy's drafting is itself a problem

## Status

The philosophy is **marked as provisional pending community co-authorship**. The pressure-test names this explicitly. A v2 with active community input is committed for within 12 months of May 2026.

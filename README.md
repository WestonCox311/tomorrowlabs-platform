# TomorrowLabs Platform

The technical implementation of TomorrowLabs's data architecture — language preservation infrastructure that bridges commercial product success and humanitarian impact.

## Status

🚧 **Pre-alpha** — Architecture is fully designed; implementation is just starting.

## What's here

- **`migrations/`** — SQL migrations defining the four-layer data architecture (~75 tables)
- **`knowledge-base/`** — Strategic context, philosophy, and architectural reasoning
- **`docs/`** — Development documentation and task tracking
- **`src/`** — Application code (Next.js, will be populated as development proceeds)
- **`scripts/`** — Migration runners, seed scripts, utilities

## Getting started

If you're picking this up fresh, read in this order:

1. `SETUP.md` — one-time setup instructions
2. `CLAUDE.md` — orientation for Claude Code
3. `knowledge-base/00-start-here/01-orientation.md` — the strategic context
4. `docs/TASKS.md` — the development task sequence
